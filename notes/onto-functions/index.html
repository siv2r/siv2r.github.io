onto-functions

Mon Jun 29 2026

Onto (surjective) A function f:A→Bf: A \to Bf:A→B is onto if for every y∈By \in By∈B there exists at least one x∈Ax \in Ax∈A with f(x)=yf(x) = yf(x)=y. Every target in the codomain gets hit, so the range fills BBB completely with no unreachable outputs.

The whole question onto answers is simple: can every output be produced? Pick any yyy you like in the codomain, and onto promises some input maps to it. Geometrically, for f:R→Rf: \mathbb{R} \to \mathbb{R}f:R→R, the graph has to touch every height on the y-axis. No gaps, no unreachable yyy.

Proving onto: run the definition backwards

The definition reads "for every yyy, there exists an xxx", so the proof always has the same four moves. Start with an arbitrary y∈By \in By∈B. Solve f(x)=yf(x) = yf(x)=y for xxx in terms of yyy. Check that this xxx actually lands in the domain AAA. Then plug it back and confirm f(x)f(x)f(x) really simplifies to yyy.

For f(x)=2x−3f(x) = 2x - 3f(x)=2x−3 on R→R\mathbb{R} \to \mathbb{R}R→R, the candidate input is x=(y+3)/2x = (y+3)/2x=(y+3)/2, which is real for any real yyy, and substituting it back gives yyy. That is the recipe in full: find the preimage, check it lives in the domain, verify it maps where you want.

Not-onto, and what it does not mean

The cheapest not-onto function is f(x)=1f(x) = 1f(x)=1 on R→R\mathbb{R} \to \mathbb{R}R→R. Every input is used, but the output is just {1}\{1\}{1}, so the rest of R\mathbb{R}R is never reached. The function f(x)=x2f(x) = x^2f(x)=x2 misses every negative number the same way. Both are perfectly valid functions, they just fail to cover their codomain.

Outputs may have gaps, inputs never can I first read "not onto" as the function being allowed to skip some inputs, leaving part of the domain unused. That mixes up the two sides. A function is total by definition: every element of its stated domain must map somewhere. If a rule is undefined at a point, that point is not in the domain (1/x1/x1/x has domain R∖{0}\mathbb{R} \setminus \{0\}R∖{0}, not R\mathbb{R}R). Onto is about the other side, whether the outputs cover BBB. Outputs are free to leave gaps, which is exactly what not-onto means. Inputs are not.

Why onto is worth proving

Two reasons. First, guaranteed solvability. Onto is an existence theorem in disguise: if fff is onto, the equation f(x)=yf(x) = yf(x)=y has a solution for every yyy. In linear algebra, a transformation T:Rn→RmT: \mathbb{R}^n \to \mathbb{R}^mT:Rn→Rm being onto is exactly the statement that Ax=bAx = bAx=b is solvable for every bbb.

Second, onto is half of what you need for an inverse. To build f−1:B→Af^{-1}: B \to Af−1:B→A, the inverse has to be defined on all of BBB, because a function is total on its domain. If fff is not onto, some y∈By \in By∈B has no preimage, so f−1f^{-1}f−1 has nothing to send it to and fails to be a function.

Onto is only half of an inverse

Onto gives every yyy at least one preimage. For an inverse you also need at most one, which is the one-to-one (injective) condition: no output is shared by two inputs, or f−1f^{-1}f−1 would have to send one yyy to two places and stop being deterministic. Onto plus one-to-one is bijective, exactly one preimage per yyy, and that is when f−1f^{-1}f−1 exists. The one-to-one side, and the trap that catches most people proving it, is its own story in one-to-one-proof-converse-trap.

Equal sizes collapse onto and one-to-one

When the domain and codomain are finite and the same size, ∣A∣=∣B∣=n|A| = |B| = n∣A∣=∣B∣=n, onto and one-to-one stop being separate properties. Prove one and the other comes free. The reason is the Pigeonhole Principle: if fff is not one-to-one, two inputs collide on one output, so the nnn inputs cover at most n−1n-1n−1 outputs, leaving some element of BBB unreached, which is not onto. Run it the other way and not-onto forces not one-to-one.

Infinite sets break this. Take f:N→Nf: \mathbb{N} \to \mathbb{N}f:N→N with f(1)=1f(1) = 1f(1)=1 and f(n)=n−1f(n) = n - 1f(n)=n−1 for n>1n > 1n>1. It is not one-to-one, since 111 and 222 both map to 111, yet it is onto, since every yyy has preimage y+1y + 1y+1. The bound that locked the two properties together is gone. This asymmetry is the very definition of an infinite set: a set is infinite exactly when it has a map to itself that is onto but not one-to-one (a Dedekind-infinite set).

The handle Onto is a claim about the output side: every target in BBB gets hit. Inputs are never allowed gaps, that is what makes fff a function. Outputs are allowed gaps, that is what makes a function not-onto, and onto closes that gap. Add one-to-one and the inverse appears. For equal finite sizes the two properties come as a pair, for infinite sets they part ways.
