The Problem

Three random variables: MMM (message), KKK (key), C=M⊕KC = M \oplus KC=M⊕K (ciphertext). I know the distributions of MMM and KKK individually. How do I find the distribution of CCC?

More generally: when a new random variable is defined as a function of others (Z=f(X,Y)Z = f(X, Y)Z=f(X,Y)), how do I compute its PMF?

Where I Got Stuck

I could reason about P(event A)P(\text{event A})P(event A) using Venn diagrams and set operations. But random variables are different. When someone writes C=K⊕MC = K \oplus MC=K⊕M and asks for P(C=1)P(C = 1)P(C=1), the answer depends on how KKK and MMM relate to each other. I didn't have a systematic way to go from "I know P(K=k)P(K = k)P(K=k) and P(M=m)P(M = m)P(M=m)" to "here's P(C=c)P(C = c)P(C=c)."

The event notation doesn't give you a way to do arithmetic on probabilities. You can intersect events, union them, complement them. But you can't XOR two events or add them. Random variables let you do that because they're functions, not sets.

Random Variables Are Functions

A random variable XXX is a deterministic function from the sample space to real numbers:

X:Ω→RX: \Omega \to \mathbb{R}X:Ω→R

P(X=k)P(X = k)P(X=k) is shorthand for the probability of the event {ω∈Ω∣X(ω)=k}\{\omega \in \Omega \mid X(\omega) = k\}{ω∈Ω∣X(ω)=k}, the inverse image of kkk under XXX.

Because RVs are functions, we can compose them. C(ω)=K(ω)⊕M(ω)C(\omega) = K(\omega) \oplus M(\omega)C(ω)=K(ω)⊕M(ω) for every outcome ω\omegaω. That's what "C=K⊕MC = K \oplus MC=K⊕M" means. The new variable CCC is just another function on the same sample space, defined pointwise.

The Joint Table Is the Workspace

With two RVs KKK and MMM, stop listing sample space outcomes. Build the joint PMF:

PK,M(k,m)=P(K=k and M=m)P_{K,M}(k, m) = P(K = k \text{ and } M = m)PK,M​(k,m)=P(K=k and M=m)

For fair independent coins (K,M∈{0,1}K, M \in \{0, 1\}K,M∈{0,1}):

If K⊥MK \perp MK⊥M: each cell is P(K=k)⋅P(M=m)P(K{=}k) \cdot P(M{=}m)P(K=k)⋅P(M=m).

This table is the object you work with. Everything else follows from it.

Marginalization: P(K=k)P(K = k)P(K=k) = sum of row kkk. You project the 2D table onto one axis.

Conditioning: P(M=m∣K=k)P(M = m \mid K = k)P(M=m∣K=k) = take row kkk, divide each cell by the row sum. Same "shrink the universe" idea from conditional-probability, applied to the table instead of Venn diagrams.

Deriving a New Distribution

To find P(C=c)P(C = c)P(C=c) where C=K⊕MC = K \oplus MC=K⊕M: look at every cell (k,m)(k, m)(k,m) in the joint table, check whether k⊕m=ck \oplus m = ck⊕m=c, and sum those probabilities.

P(C=c)=∑k,mk⊕m=cP(K=k,M=m)P(C = c) = \sum_{\substack{k, m \\ k \oplus m = c}} P(K{=}k, M{=}m)P(C=c)=∑k,mk⊕m=c​​P(K=k,M=m)

For our fair coins: P(C=1)P(C = 1)P(C=1) comes from cells where k≠mk \neq mk=m, which are (0,1)(0, 1)(0,1) and (1,0)(1, 0)(1,0). So P(C=1)=0.25+0.25=0.5P(C = 1) = 0.25 + 0.25 = 0.5P(C=1)=0.25+0.25=0.5.

The same technique works for any operation. For addition (Z=X+YZ = X + YZ=X+Y) with independent RVs, fixing X=kX = kX=k forces Y=z−kY = z - kY=z−k, giving the convolution formula:

P(Z=z)=∑kP(X=k)⋅P(Y=z−k)P(Z = z) = \sum_{k} P(X = k) \cdot P(Y = z - k)P(Z=z)=∑k​P(X=k)⋅P(Y=z−k)

If XXX and YYY are dependent, replace P(Y=z−k)P(Y = z - k)P(Y=z−k) with P(Y=z−k∣X=k)P(Y = z - k \mid X = k)P(Y=z−k∣X=k).

That's the general technique: enumerate all input pairs that produce the target output, sum their joint probabilities.

XOR vs Addition: Why the Outputs Look Different

The operation you apply determines the shape of the resulting distribution.

XOR is a permutation. Fix any row kkk in the joint table. As mmm ranges over {0,1}\{0, 1\}{0,1}, the output k⊕mk \oplus mk⊕m hits every value exactly once. Every row is a permutation of the output space. If KKK is uniform, each output value gets equal total weight.

Addition piles up in the middle. Multiple input pairs can produce the same sum. Middle values have more combinations (4=1+3=2+2=3+14 = 1{+}3 = 2{+}2 = 3{+}14=1+3=2+2=3+1) while edge values have fewer (2=1+12 = 1{+}12=1+1). Mass accumulates in the center.

Adding more independent uniform variables keeps smoothing the peak toward a bell curve (Central Limit Theorem).

The Row-Universe Analysis

This is the part that connects to the one-time pad. Split the joint table by rows of KKK. Each row is a separate universe conditioned on a specific key value.

  • Row K=0K = 0K=0: C=0⊕M=MC = 0 \oplus M = MC=0⊕M=M. The ciphertext copies the message.
  • Row K=1K = 1K=1: C=1⊕M=M‾C = 1 \oplus M = \overline{M}C=1⊕M=M. The ciphertext flips the message.

For CCC to be independent of KKK, the distribution of CCC must look the same in every row:

P(C=1∣K=0)=P(C=1∣K=1)P(C = 1 \mid K = 0) = P(C = 1 \mid K = 1)P(C=1∣K=0)=P(C=1∣K=1)

The left side is P(M=1)P(M = 1)P(M=1). The right side is P(M=0)P(M = 0)P(M=0). These are equal only when P(M=1)=P(M=0)=1/2P(M = 1) = P(M = 0) = 1/2P(M=1)=P(M=0)=1/2, so MMM must be uniform.

This is the one-time pad property: Any bias ⊕\oplus⊕ Uniform === Uniform. Even if MMM is 90% zeros, a uniform KKK washes it away completely:

P(C=c)=∑mP(M=m)⋅P(K=c⊕m)=∑mP(M=m)⋅1N=1NP(C = c) = \sum_{m} P(M = m) \cdot P(K = c \oplus m) = \sum_{m} P(M = m) \cdot \frac{1}{N} = \frac{1}{N}P(C=c)=∑m​P(M=m)⋅P(K=c⊕m)=∑m​P(M=m)⋅N1​=N1​

The 1/N1/N1/N factors out of the sum, and the remaining ∑mP(M=m)=1\sum_m P(M = m) = 1∑m​P(M=m)=1.

Independence Matters, Not Just Uniformity

KKK being uniform isn't enough. KKK must also be independent of MMM.

Counterexample: let K=MK = MK=M (perfect dependence). KKK is still uniform since MMM is fair. But C=M⊕K=M⊕M=0C = M \oplus K = M \oplus M = 0C=M⊕K=M⊕M=0 always. The ciphertext is a constant. It leaks everything about MMM (namely, that M=KM = KM=K).

The proof above used P(K=c⊕m)P(K = c \oplus m)P(K=c⊕m) without conditioning on M=mM = mM=m. That step requires K⊥MK \perp MK⊥M. Without independence, you'd need P(K=c⊕m∣M=m)P(K = c \oplus m \mid M = m)P(K=c⊕m∣M=m), and you can't conclude uniformity.
